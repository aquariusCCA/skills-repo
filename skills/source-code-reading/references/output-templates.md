# 原始碼閱讀輸出模板

只有在使用者要求結構化筆記、任務範圍較廣，或使用者未指定輸出格式且可重用筆記會有幫助時，才使用這些模板。

各段落保持精簡。不適用的段落可以省略。

除非使用者明確要求超短答，輸出至少應包含：

- 範圍與入口
- 已閱讀檔案
- 已確認行為
- 待驗證
- 下一步閱讀或下一步行動

若任務涉及 monorepo、多服務或跨工作區流程，應在範圍中明確標示 `project_boundary` 或跨界點。

## 跨服務 / 跨工作區流程模式

適合：流程會跨 app、service、workspace 或明顯邊界。

```markdown

- 模式：feature

- `path/to/file`：此檔案在跨界流程中的角色

## 跨界流程摘要

1. 入口如何觸發第一段流程。
2. 在哪裡跨到下一個 app / service / workspace。
```
