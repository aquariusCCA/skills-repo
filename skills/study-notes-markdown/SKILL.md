# Study Notes Markdown

當使用者希望保留教材型、教學導向、較詳細的正文內容，但又希望筆記在後續更容易理解、瀏覽、搜尋、跳讀、回查與複習時，使用這個 skill。

這個 skill 的核心不是把內容濃縮成摘要，而是讓原本的筆記「更好用」。

---

# 如何使用這個 skill

使用者不需要在 Markdown 檔案中加入特殊語法，也不需要在終端執行指令。
當使用者要求整理某個學習筆記檔案，且提到 `study-notes-markdown` 或描述符合本 skill 的用途時，Codex 應讀取本 skill 並依照其規則處理目標檔案。

典型請求：

```text
請用 study-notes-markdown 整理 `XXX.md`
```

---

# 核心目標

不要刪掉核心教學內容。  
不要把詳細說明改成只有提綱。  
不要為了整齊而犧牲理解。
不要為了複習方便而只留下結論。

重點是替既有內容補上穩定的外層結構，讓筆記同時支援：

- 第一次學習時可順讀
- 複習時可快速定位
- 搜尋時容易命中
- 跳讀時不容易迷路
- 回查時能快速判斷是否找對頁
- 之後擴充時維持一致性

---

# 這個 skill 要解決的問題

常見原始筆記問題包括：

- 單篇過長，讀到中途容易失去定位
- 缺乏章節導航，不知道先看哪篇
- 回查時只能全文搜尋，效率低
- 筆記保留了很多內容，但不容易快速抓到入口
- 內容本身不一定差，但結構不足，導致難讀、難找、難複習

這個 skill 的工作重點是補結構、補入口、補導航、補回查線索。
不是把所有內容都重寫成精簡版。

---

# 成功標準

整理完成後，筆記應盡量達成以下效果：

1. 使用者一打開就知道這篇在講什麼
2. 使用者知道這篇適合怎麼讀
3. 使用者知道這篇和前後知識點的關係
4. 使用者之後想回查時，能在短時間內找到答案
5. 使用者複習時，不必重讀整篇才能抓到重點位置
6. 使用者在倉庫中不容易迷路
7. 原本重要的定義、前提、限制、例外與推導沒有因整理而消失
8. 模板化後，內容仍保有原本應有的表述精度

# 核心原則

## 1. 保留教材型正文

- 除非使用者明確要求重寫，否則保留原本教學型正文
- 不用摘要取代正文
- 不把段落改成只剩大綱式速記
- 允許做的是加上外層結構、導航、標示與少量輔助

## 2. 先補結構，再碰內容

整理順序優先是：

1. 倉庫入口與章節導航
2. 先判斷是否需要拆分大筆記
3. 單篇筆記開頭格式統一
4. 必要的內部連結
5. 輕量型複習輔助
6. 最後才是小幅內容微調

## 3. 小改穩改，不過度濃縮

- 優先做可重複套用的結構化修改
- 不做大量語意重寫
- 不追求把每篇都整理成考前速記
- 不追求每段都加總結，避免破壞原文節奏
- 不以壓縮字數作為整理成功標準

## 4. 理解優先於整齊

若形式上的整齊會讓內容更難懂，應保留原本較易理解的寫法。  
整理的目的是讓筆記更容易吸收，而不是只讓版面變漂亮。

## 5. 模板是輔助，不是硬格式

- 模板可以套用，但不能機械式硬套
- 可依主題、內容密度、長度、用途微調
- 若模板會破壞內容的閱讀流暢度，可調整或省略局部欄位
- 不應為了欄位完整，而強行改壞原文節奏

## 6. 結構一致性要服務於使用，而不是反過來支配內容

- 一致性很重要，但應服務於閱讀、查找與回查
- 如果同一種模板套在所有筆記上反而造成干擾，應允許差異化處理

---

# 優先級規則

當多個目標發生衝突時，優先順序如下：

1. 理解
2. 表述精度
3. 回查效率
4. 結構一致性
5. 版面整齊

若模板化會傷害理解或精度，應允許偏離模板。

---

# 工作流程

1. 檢查整體倉庫結構與章節分布
2. 判斷目前是大筆記還是小筆記
3. 若為大筆記，先拆分，再整理拆分後的小筆記
4. 若為小筆記，直接整理
5. 先補 README / 導航 / 開頭格式，並檢查根目錄 `README.md` 的目前進度與章節入口是否需要同步
6. 再補高價值內部連結
7. 最後再視需要補複習輔助
8. 最後檢查是否因整理而損失表述精度

## 執行上的硬規則

- 若已判定為大筆記，不要只補文首導讀就結束；應先完成拆分，再整理拆分後的檔案
- 若同時符合大筆記條件與小筆記條件，優先按大筆記處理
- 若最終決定不拆，必須明確說明不拆的理由，例如篇幅其實可控、主題高度集中、拆分反而破壞理解
- 若拆分後會產生一組子筆記，應建立對應入口頁 `README.md`，而不是只散落多個檔案

---

# 大筆記與小筆記的判斷

## 視為大筆記的情況

符合以下任一情況時，應視為大筆記並先拆分：

- 單篇過長，閱讀與搜尋成本高
- 同時涵蓋多個明顯子主題
- 已有穩定的標題階層，例如 `#`、`##`、`###`
- 使用者在單篇中不容易快速定位內容
- 單篇已同時承擔總覽、定義、比較、實作、例子等多種任務

### 建議的判斷門檻

符合以下任一門檻時，預設應拆分，除非有明確理由證明不拆更好：

- 單篇超過 `1500` 行
- 單篇有 `5` 個以上一級主題，或 `10` 個以上二級主題
- 同一篇同時包含「概念講解」與「多個獨立案例 / 題解 / 實作主題」
- 單篇已經接近一本小冊，讀者難以靠搜尋與標題快速定位

## 視為小筆記的情況

符合以下情況時，可直接整理：

- 單篇主題明確且範圍集中
- 長度可控
- 適合獨立閱讀與獨立回查
- 不拆分也不影響理解與維護

### 小筆記的補充判準

- 通常可在一次閱讀中消化，不需要頻繁上下跳讀
- 即使補上導讀、關鍵字與少量導航後，也已足夠支援回查
- 拆分後不會明顯增加理解收益，反而可能造成內容過碎

## 拆分時的實作規則

- 先沿用原本穩定的標題階層拆分，不要為拆而拆
- 優先拆成「主題資料夾 + README + 多篇小筆記」的形式
- 拆分資料夾預設使用原筆記檔名去掉 `.md` 後的名稱，例如 `2 链表.md` 拆成 `2 链表/`
- 拆分後的入口頁應放在該資料夾的 `README.md`
- 若原筆記已經有章節層級的 `images/` 目錄，拆分後的小筆記預設沿用原圖片，不複製資產
- 每篇拆出的子筆記都應能獨立閱讀，並保有必要的上下文
- 拆分後的檔名應具可回查性，例如 `01 概述.md`、`02 单向链表.md`、`06 LeetCode 206 反转链表.md`
- 拆分後應補上前後閱讀順序與返回入口連結，避免讀者從單篇進入後迷路

### 拆分資料夾命名的例外

- 若原檔名過於泛化，例如 `README.md`、`笔记.md`、`总结.md`，可改用更具主題性的資料夾名
- 若同名資料夾已存在，先判斷是否本來就是這篇筆記的拆分目錄；若不是，應改用更清楚且不衝突的名稱
- 若原檔名已帶有明確序號，拆分後應保留該序號，避免破壞原本閱讀順序

# 根目錄 README

讓根目錄 `README.md` 成為整份筆記的入口頁，而不是只放標題。

根目錄 README 應優先包含：

1. 倉庫用途簡介
2. 目前學習進度
3. 章節導航
4. 如何使用這份筆記
5. 需要快速回查的入口
6. 參考資料

## 根目錄 README 建議內容

- 這份筆記在學什麼
- 目前已整理到哪裡
- 各章節連結
- 建議閱讀順序
- 適合初學順讀的章節
- 適合複習跳讀的章節
- 參考資料放在導航之後，而不是之前

---

# 章節 README

每個主要章節資料夾都應有自己的 `README.md`。

章節 README 應包含：

- 本章主要內容
- 本章和整體學習地圖的關係
- 建議閱讀順序
- 本章所有小節連結
- 哪些主題適合快速回查
- 返回根目錄 README 的連結

## 章節 README 的用途

章節 README 不只是列表頁，它應該回答：

- 這章在學什麼
- 先讀哪篇比較不會卡
- 哪幾篇是概念基礎
- 哪幾篇是常回查頁
- 這章讀完後通常接哪一章

# README 同步規則

整理單篇筆記時，應檢查該筆記所在章節的 `README.md`：

- 若章節 `README.md` 不存在，且單篇筆記使用了 `所属章节：[章節名](./README.md)`，應建立章節 `README.md`
- 若章節 `README.md` 已存在，但未包含本篇筆記，應同步補上本篇連結
- 若整理後改動了標題、檔名、閱讀順序、延伸閱讀或回查入口，應同步更新章節 `README.md`
- 若章節 `README.md` 在 Git 狀態中顯示為刪除，應明確說明此狀態；若整理後會造成壞鏈，應重建或請使用者確認

整理單篇筆記時，也應檢查根目錄 `README.md` 的目前進度、章節導航與快速入口是否仍準確。
根目錄 `README.md` 不需要每次整理單篇都機械式改動，但只要符合以下情況，就必須同步：

- 新增、刪除、移動章節資料夾
- 新增章節 `README.md`
- 章節導航或整體學習路徑發生變化
- 根目錄 README 的章節入口缺失或已失效
- 根目錄 README 的目前進度與已整理小節不一致
- 根目錄 README 未列出已存在且應作為入口的章節或小節
- 單篇整理後新增、移動、改名或補齊章節入口，導致總目錄需要更新才能避免讀者迷路

若檢查後決定不更新根目錄 `README.md`，應能說明理由，例如總目錄已包含該章入口、目前進度仍正確，且本次整理沒有改變章節導航或整體學習路徑。

# 單篇筆記開頭格式

每篇筆記應優先補齊以下開頭結構：

1. 標題
2. 所属章节
3. 本节导读
4. 关键字
5. 建議回查情境
6. 正文教學內容
