# study-notes-markdown prompts

這份檔案提供常用提示詞範例，讓使用者更容易明確喚起 `study-notes-markdown`。

重點不是要求模型重寫內容，而是要求它保留教材型正文，補上結構、導航、查找入口與複習支架。

---

## 最小請求

請用 `study-notes-markdown` 整理 `XXX.md`

## 保留正文

請用 `study-notes-markdown` 整理 `XXX.md`，保留原本教材型正文，不要改成摘要或大綱速記

## 拆分大筆記

請用 `study-notes-markdown` 整理 `XXX.md`，若內容已經是大筆記，請先拆分成「主題資料夾 + README + 多篇小筆記」，再補導航與前後連結

## 單篇整理

請用 `study-notes-markdown` 整理 `XXX.md`，補上標題、本節導讀、關鍵字、建議回查情境，必要時補 30 秒複習入口與速查區，但不要刪掉正文

## 補查找入口

請用 `study-notes-markdown` 整理 `XXX.md`，重點加強查找能力：補同義詞、英文術語、常見搜尋詞、FAQ 或索引入口，讓之後比較容易回查

## 補複習支架

請用 `study-notes-markdown` 整理 `XXX.md`，保留正文，並補上建議回查情境、30 秒複習入口、速查區與必要的自測問題

## 章節整理

請用 `study-notes-markdown` 整理 `某章資料夾/`，補章節 `README.md`、建議閱讀順序、高頻回查入口、易混淆主題與返回根目錄的導航

## 整份倉庫整理

請用 `study-notes-markdown` 整理整份筆記倉庫，先檢查根目錄 `README.md`、章節入口與目前進度，再逐步整理單篇筆記；如果有大筆記，先拆分再整理

## README 同步

請用 `study-notes-markdown` 整理 `XXX.md`，如果這次整理會影響章節入口、檔名、閱讀順序或快速回查入口，請同步更新章節 `README.md` 和根目錄 `README.md`

## 正文版加複習版

請用 `study-notes-markdown` 整理 `XXX.md`，保留正文版本，另外再產出一份複習版，適合考前快速回看，但不要把正文直接壓縮成小抄
