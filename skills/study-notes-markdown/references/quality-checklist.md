# 品質檢查清單

大量整理、多檔改動、準備收尾驗收，或懷疑模板化過度時，讀取本檔。

---

## 理解

- 一打開就知道這篇在講什麼
- 知道這篇適合何時讀、何時回查
- 原本重要的定義、前提、限制、例外與推導沒有消失

## 閱讀

- 倉庫、章節、單篇三層入口清楚
- 導讀、閱讀順序與延伸閱讀能降低迷路機率
- 大筆記若已明顯過長，是否已先拆分而非只補表面欄位

## 查找

- 是否補齊關鍵字、別名、英文術語與常見搜尋詞
- 是否有高頻查找主題入口、FAQ、索引或足夠的 README 導航
- 檔名、標題與連結是否真的有助於回查

## 複習

- 是否具備穩定的回查情境與複習入口
- 速查區是否收納高頻回查項，而非複製整段正文
- 自測問題是否對應正文內容，而不是另外發明知識點

## 結構

- 根目錄 README、章節 README、單篇入口是否同步
- 是否留下壞鏈、孤兒檔案、無入口的新檔案
- 是否保留原倉庫語言風格與命名慣例

---

## 反模式

- 把正文壓縮成只有提綱或考前小抄
- 為了整齊硬塞所有欄位，結果破壞閱讀節奏
- 補了很多標題，但沒有真正提升導航、搜尋或回查能力
- FAQ、速查區、正文三者互相重複，且內容不一致
- 新增了拆分檔案，卻沒有入口 README 或返回連結
- 只改單篇，不同步章節與根目錄入口，導致讀者迷路
- 看不到完整倉庫時，卻聲稱已完成全域同步與壞鏈驗證

---

## 收尾輸出

完成整理後，應能簡潔說明：

- 本次整理的範圍
- 是否做了拆分、重命名、README 同步
- 是否補了查找系統或複習支架
- 是否仍有未處理的壞鏈、缺口或需人工決策的部分

若本次為大量整理、多檔改動或正式收尾，可使用以下格式：

### 本次整理範圍

- 整理的是：整份倉庫 / 章節 / 單篇 / 大筆記拆分組
- 實際處理的檔案或目錄：

### 結構處理

- 是否拆分：是 / 否
- 若未拆分，理由是：
- 是否新增或更新 `README.md`：根目錄 / 章節 / 拆分入口 / 無
- 是否有重命名、移動檔案或調整閱讀順序：

### 查找與複習補強

- 是否補關鍵字系統：
- 是否補 FAQ / 索引 / 術語表 / 內部連結：
- 是否補 `30 秒複習入口` / `速查區` / `自測問題`：

### 驗收結果

- 是否已檢查壞鏈：已檢查 / 無法完整驗證
- 是否已同步章節與根目錄入口：已同步 / 部分同步 / 無法驗證
- 是否保留正文表述精度：是 / 否，若否請說明

### 待確認或未處理項目

- 缺失檔案
- 無法驗證的連結
- 需要人工決策的命名或拆分問題
- 其他限制
