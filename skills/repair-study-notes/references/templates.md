# 模板

把這些當作輸出骨架與寫法參考，不要機械式硬套。真正的判斷邏輯、適用情況、限制事項，以 `SKILL.md` 為準。

## 目錄

- 單篇筆記重構模板
- 原文主要問題摘要模板
- 本次修正決策模板
- 錯誤與矛盾處理模板
- 原文問題說明模板

## 錯誤與矛盾處理模板

### 修正說明

適用於可明確確定原文有誤時。

```md
## 修正說明

- 原文此處將＿＿＿＿說成＿＿＿＿，這裡改為＿＿＿＿，因為＿＿＿＿。
```

### 待查證

適用於可能錯誤，但尚未能完全確認時。

```md
## 待查證

- 原文此處指出＿＿＿＿，但依上下文 / 常見定義 / 例子來看，這裡可能有誤，需再查證。
```

### 術語混用

適用於同一概念在原文中出現多個名稱時。

```md
下文統一使用「＿＿＿＿」這個名稱，原文中出現的「＿＿＿＿ / ＿＿＿＿」在此視為同一概念。
```

### 衝突處理

適用於原文前後說法互相衝突時。

```md
## 衝突處理

- 原文前段認為＿＿＿＿，後段又認為＿＿＿＿，兩者彼此衝突。
- 目前依＿＿＿＿作為主判準，以下正文採用＿＿＿＿版本。
- 另一種說法暫列為待確認。
```

## 原文問題說明模板

當原文問題本身對學習有價值時，可以保留一小節：

```md
## 原文容易誤解的地方

原文之所以難懂，通常是因為：

1. 先講了結果，後講條件
2. 同一名詞前後用法不一致
3. 例子沒有明確對應到規則
```

但這一節不能喧賓奪主，重點仍然是給出整理後的穩定版本。
